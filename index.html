      plain text

      No special behavior. Whatever you type is exactly what gets rendered.

      ACS string escapes

      More or less compatible with ZDoom's ACS Print function.

      The main use for this is to easily change colors with \c sequences, which are listed in the color table.

      Full list of supported escapes:

      • \cX ‚Äî Set the color, using a letter to identify a standard translation.
      • \c[NAME] ‚Äî Set the color by name.
      • \x00 ‚Äî Insert a Latin-1 character using a two-digit hex code.
      • \000 ‚Äî Insert a Latin-1 character using a three-digit octal code. This is some ancient C nonsense. Don't use it.
      • \\ ‚Äî Insert a literal backslash.
      • \n ‚Äî Insert a line break.
      • \" ‚Äî Insert a quotation mark.

      bbcode

      Uses HTML-like tags, with square brackets, like so: [font=doom menu]Ultra-Violence[/font].

      Tags don't need to be closed (they're assumed to all close at the end of the message), but if you do close them, they have to nest correctly ‚Äî that is, close the innermost tag first.

      Use [[ or ]] for a literal square bracket.

      Supported tags:

      • [font=...] ‚Äî Set the font. Fonts are identified by their display names, and the first font that contains all the given words is chosen. For example, [font=jen b] will select Jenocide Big.
